# Domain Language

프로젝트 도메인 언어(유비쿼터스 언어) **인덱스**. 용어 정의 본문은 `.claude/domain/` 하위 도메인 파일에 둔다.

해당 도메인 작업(코드·주석·테스트 제목·커밋 메시지·UI 문구 작성) 시 아래 도메인 파일을 먼저 읽고 용어를 따른다.

> **목적**: 사람과 에이전트가 같은 용어를 같은 의미로 쓰게 하여, 팀원·설정·세션 간 용어 모호성을 없앤다.

## 새 도메인 파일 작성

1. `.claude/domain/_TEMPLATE.md`를 복사해 `.claude/domain/{domain}.md`로 만든다.
2. 템플릿의 작성 규칙에 따라 용어를 채운다.
3. 위 **도메인 인덱스**에 한 줄을 추가한다 (대표 용어 요약 → 파일 경로).

## 유지보수

- 인덱스에는 용어 정의를 적지 않는다. 정의는 도메인 파일에만 둔다.
- 기능 계획 중 용어가 새로 생기거나 충돌하면 해당 도메인 파일의 정의와 `Flagged ambiguities`를 갱신한다.
- 도메인 파일은 **프로젝트 고유 지식**이다. 팀 표준 적용기(`/team-init`)는 이 인덱스와 `_TEMPLATE.md`만 제공하며, 실제 도메인 정의 파일은 건드리지 않는다.
