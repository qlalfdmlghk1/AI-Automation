# Team Claude Standard v2.0.0

## 1. 핵심 작업 원칙

의미 있는 변경은 Plan → Code → Review 순서로 진행합니다.

Plan 단계에는 다음을 포함합니다.

- 변경 목적
- 변경 범위
- 접근 방식
- 검증 계획
- 미해결 질문

Code 단계에서는 기존 코드 스타일과 프로젝트 구조를 우선합니다.

Review 단계에서는 위험 포인트, 테스트 포인트, 후속 작업을 명확히 남깁니다.

## 2. 언어와 응답

- 기본 응답 언어는 한국어입니다.
- 코드, 파일명, 타입, 명령어는 영어를 유지합니다.
- 사용자가 거부한 접근 방식은 같은 세션에서 다시 제안하지 않습니다.

## 3. 작업 보호 장치

- 요청 범위 밖의 변경은 자동 수정하지 않고 알림으로 분리합니다.
- 새로운 라이브러리 추가 전 기존 코드와 유틸 재사용을 우선 검토합니다.
- 인증, 권한, 개인정보, 결제, 배포 설정 변경은 사용자 확인 후 진행합니다.
- 실제 env 값, 토큰, 쿠키, 고객 식별 정보는 문서와 로그에 남기지 않습니다.

## 4. 문서화 원칙

- 개발 시작: `docs/features/{feature}/plan.md`
- 진행 기록: `docs/features/{feature}/progress.md`
- 빠른 히스토리: `/note`
- 기술 문서: `/tech-doc`로 Confluence 초안 생성
- Jira, GitHub, Confluence는 링크로 연결합니다.
